Technical Overview

Enterprise Research Assistant
How It Works

A deep dive into the multi-agent AI pipeline — from the moment you type a question to the cited, evaluated answer you receive.

Built with LangGraph · GPT-4o · Pinecone · Tavily · FastAPI · React

1. The Big Picture

When you type a research question and hit Send, your query doesn't go to a single AI model. It flows through a pipeline of six specialized AI agents, each with a different job. Think of it like a small research team:

🧠 The Manager (Planner)

Reads your question and decides what research tasks need to be done — like a team lead assigning work.

🔍 The Researchers (RAG Retriever + Web Searcher + Code Executor)

Three workers run in parallel — one searches the internal knowledge base, one searches the live web, one runs code if needed — like junior researchers going off to gather information.

✍️ The Writer (Synthesizer)

Takes everything the researchers found and writes one clear, cohesive, cited answer — like a senior writer combining all research notes into a final report.

⚖️ The Fact-Checker (Evaluator)

Reads the answer and scores it on faithfulness and relevancy before it ever reaches you — like an editor who won't let bad work go to print.

2. Step-by-Step Flow

  1. You type a question Your question hits the FastAPI backend at /query. Before anything else, the Guardrails layer checks for prompt injection attacks (attempts to hijack the AI). If clean, it proceeds.
  2. The Planner breaks it down GPT-4o reads your question and produces a structured research plan — identifying what sub-questions need answering and what tools are needed (RAG, web search, code execution).
  3. Three agents gather information in parallel RAG Retriever queries Pinecone (a vector database) for relevant pre-indexed documents using semantic similarity search. Web Searcher calls the Tavily API for live, current web results. Code Executor runs Python code if the question requires computation. All three use the cheaper GPT-3.5 model to keep costs low.
  4. The Synthesizer writes the answer GPT-4o combines all gathered information into one coherent answer, inserting numbered citations [1], [2], [3] inline — mapped to the actual sources (URLs, document names).
  5. The Evaluator fact-checks it GPT-4o acts as an impartial judge and scores the answer on two dimensions: Faithfulness (does the answer stick to what the sources actually say?) and Relevancy (does it answer what was asked?). Scores below 0.7 trigger a retry loop.
  6. The answer reaches you The final answer arrives in the UI with clickable citation cards, latency, cost in dollars, and token count — all from the real API call.

4. The Smart Cost Routing

Running every agent on GPT-4o would be expensive. The system uses cost-aware model routing — a key architectural decision that reduces cost per query by ~62%:

AgentModel UsedWhy
PlannerGPT-4oNeeds deep reasoning to understand complex questions
RAG RetrieverGPT-3.5-turboJust processing retrieved text chunks — no heavy reasoning needed
Web SearcherGPT-3.5-turboParsing and filtering web results — straightforward task
Code ExecutorGPT-3.5-turboShort, structured code generation — simple task
SynthesizerGPT-4oMust write a coherent, well-cited response — quality matters
EvaluatorGPT-4oMust judge quality accurately — needs the best model
"Use a Ferrari where it counts, a Honda where it doesn't." — That's the routing philosophy.

5. RAG — The Internal Knowledge Base

RAG stands for Retrieval-Augmented Generation. Instead of relying only on what GPT-4o was trained on (data up to a cutoff date), the system maintains its own knowledge base in Pinecone — a vector database.

Here's how it works:

  1. Documents are embedded Each document in the knowledge base is converted into a vector (a list of 1,536 numbers that represent its meaning) using OpenAI's embedding model.
  2. Your question is also embedded When you ask a question, it gets converted to a vector in the same space.
  3. Cosine similarity finds the best matches Pinecone finds the 5 documents whose vectors are most "similar" to your question vector — documents that are semantically close, not just keyword-matched.
  4. These are handed to the Synthesizer The retrieved document chunks become numbered sources [1], [2], etc. in the final answer.

6. The Evaluation System (LLM-as-a-Judge)

Most AI systems just generate an answer and send it. This system evaluates every answer before you see it using a separate GPT-4o instance acting as a judge.

MetricWhat it measuresThreshold
FaithfulnessIs the answer grounded in the retrieved sources? Does it avoid hallucination?≥ 0.87
RelevancyDoes the answer actually address what was asked?≥ 0.84

If either score is too low, the graph can loop back and attempt a better answer. The scores are returned with every API response so you can see them in the Metrics page.

7. The Guardrails System

Before any agent processes your query, it passes through three safety checks:

1. Prompt Injection Detection

Checks for patterns like "ignore previous instructions", "you are now", "disregard" — attempts by malicious inputs to hijack the AI agents. Blocked immediately with a 400 error.

2. Token Budget Enforcement

Each query has a maximum token budget (default 8,000 tokens). If an agent would exceed it, the pipeline stops early and returns whatever it has. Prevents runaway costs.

3. Tool Whitelist

Agents can only use pre-approved tools (Pinecone search, Tavily search, Python sandbox). No agent can make arbitrary external calls.

9. What Happens in the UI When You Send a Query

  1. Query sent to backend Frontend calls POST /query. The backend starts the LangGraph pipeline.
  2. Live pipeline card appears in chat A mini flow diagram shows in the chat area. Each agent node lights up with a purple spinning ring while running, turns green with a checkmark when done.
  3. Answer appears When the backend responds (~3–5 seconds), the pipeline card disappears and the answer slides in with inline citation markers [1][2][3].
  4. Citations expand Click "Citations (N)" to reveal source cards — each with title, URL, and a snippet of the source text. Clicking opens the source directly.
  5. Metadata shown Below every answer: latency (e.g. 3.1s), cost (e.g. $0.0041), and token count (e.g. 2,184 tok).
